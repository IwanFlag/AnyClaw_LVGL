## 阶段三：Bug 检查与修复

> 每完成一个功能模块后，立即进入 Bug 检查流程。

### Bug 检查流程

```
完成模块开发
  → 编译通过（bash build/linux/build.sh）
  → Wine/真机测试
  → 对照 PRD 验收标准逐条检查
  → 对照 Design UI 线框检查像素级对齐
  → 检查边界情况（窗口缩小/放大/DPI 变化/离线/超时）
  → 发现 Bug → 记录到下方 Bug 清单 → 修复 → 重新验证
  → 无 Bug → 标记模块 ✅ → 下一个模块
```
