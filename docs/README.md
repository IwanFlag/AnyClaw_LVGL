## 文档清单

| 文件 | 用途 | 内容 |
|------|------|------|
| `PRD.md` | 产品需求 | 做什么、为什么做、验收标准（13 章，纯逻辑） |
| `Design.md` | 设计规范 | 长什么样、怎么画、画在哪（22 章，4 层结构） |

**引用规则：** PRD 每节末尾引用 Design UI 编号（如 `> 设计 → UI-17`），Design 每节开头引用 PRD 功能编号（如 `> 需求 → CI-01`）。锚点用功能编号，不用章节号。

## 文档维护

- PRD 和 Design **版本号必须同步**
- 功能变更以 **PRD 为准**，Design 跟进
- 每个 Sprint 结束对照 PRD 状态标记更新
- 修改前查 Design 开头的「编号索引表」
- 运行 `scripts/verify-docs.sh` 验证一致性
